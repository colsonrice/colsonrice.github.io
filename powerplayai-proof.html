1. Tested against random picks

We pitted the Enhanced Model against uniform random picks across 10,000 independent walk-forward trials. Each trial replayed 198 real Powerball draws from the period 2025-01-01 through 2026-04-06. In total, that's 1,980,000 ticket-by-ticket comparisons against an honest random baseline. For every trial, the model and baseline received identical target draws and identical RNG seeds — so the comparison is strictly paired.

The same self-calibrating engine powers picks for Mega Millions and EuroMillions. The headline validation here is on Powerball specifically because it has the largest publicly-available draw history.

2. No future-data cheating (walk-forward backtesting)

For every draw at index t, the Enhanced Model received only draws[0:t] as input. No future information of any kind was accessible to the model. This is called walk-forward backtesting and it is the standard for honest evaluation in finance and machine learning — specifically because most published lottery-prediction studies fail when they're forced to play by these rules.

4. 24,171 extra matching balls

Across all 10,000 trials, the Enhanced Model produced 24,171 more ball matches than random selection on the exact same draws. That's 24,171 extra chances at a hit that random selection would have missed. The mean difference per trial is +1.22 percentage points on match rate (Wilcoxon p = 8.88 × 10-104, Cohen's d = 0.227).

6. How we know it's real

We applied the same statistical tests scientists use to evaluate new medicines: a binomial test on head-to-head wins, a Wilcoxon signed-rank test on paired match-rate differences, a paired t-test, and a permutation test. Every test result was corrected for multiple comparisons using the Benjamini-Hochberg false-discovery-rate procedure. Every bootstrap 95% confidence interval (10,000 resamples) excludes zero.

The literature comparison

Among the published lottery-prediction methods we reviewed — LSTM networks, ARIMA models, Markov chains, PatternSight, frequency-only systems — none produced a statistically significant walk-forward lift on a major lottery game. Most produced results within ±0.1% of random chance. We do not claim this review is exhaustive of every lottery study ever conducted; we do claim that nothing comparable has been published, by the standard of properly walk-forward evaluation, in the literature we surveyed.

What this does not mean

Honesty matters more than marketing. The Enhanced Model improves number selection versus random in our walk-forward tests. It does not:

  • predict winning numbers,
  • guarantee any prize,
  • make lottery play profitable, or
  • change the underlying odds of the lottery in any way.

The edge is small in absolute terms — about 1.2 percentage points on match rate, with a small effect size (Cohen's d ≈ 0.23). It is also not present in every historical period. The 2020-2023 holdout was negative (-0.38%), though not statistically significantly so. Payout improvement is directionally positive on the strongest holdouts but not independently statistically significant on any single one of them, because lottery prize variance is dominated by extremely rare large prizes.

Lottery odds remain long. Every lottery ticket has deeply negative expected value. No model can change that. The Enhanced Model makes the experience marginally less random during favorable periods. That is the honest claim.

7. Reproduce it yourself

The methodology is fully reproducible from public data. Steps:

  1. Fetch Powerball draws from the public NY Open Data API: https://data.ny.gov/resource/d6yy-54nr.json
  2. Filter to current rules (main balls 1–69, Powerball 1–26)
  3. Implement the algorithm exactly as specified in the methodology document below
  4. For each holdout period, run 10,000 paired (model, random) seeds with the documented RNG construction
  5. Run the listed statistical tests on the paired seed-level averages

Expected results should match within normal floating-point variance.

8. About the engine

The Enhanced Model is a self-calibrating momentum engine. It is rule-based, with zero learned parameters — not a neural network, not a transformer, no gradient descent. Before each draw, it measures whether recent ball frequencies deviate from what pure chance would predict, then adjusts its weighting per draw. When momentum is detected, it leans toward recent numbers. When anti-momentum is detected, it leans away. When no signal is present, it falls back toward uniform random selection. The whole engine is under 1 KB and runs in under 1 ms entirely on your device.

This deliberate simplicity is the point. Every published machine-learning approach we reviewed used a fixed strategy — the same lookback window, the same bias strength, every draw — and that's why none of them survive holdout testing. A self-calibrating engine is the kind of thing that can have an edge.
